## Context

CLI 当前在正式环境会从 GitHub 拉取模板/代码，导致 e2e 测试在本地无法稳定复现真实流程。需要在不影响生产行为的前提下，为 e2e 提供可控的本地拉取路径与隔离环境。

## Goals / Non-Goals

**Goals:**
- 本地与 CI 均可运行的 CLI e2e 测试，覆盖关键用户路径
- 通过显式的测试配置切换拉取来源，保持生产默认行为不变
- 统一测试夹具与隔离策略，确保可重复与可清理

**Non-Goals:**
- 重写 CLI 现有核心逻辑或引入新的模板格式
- 改变发布流程或生产环境的默认拉取来源

## Decisions

- 通过环境变量或 CLI 选项提供“拉取来源覆盖”，默认仍为 GitHub，测试中使用本地路径。这样保持向后兼容且易于在测试里控制。
- e2e 测试使用隔离的临时目录与固定夹具仓库，避免污染开发环境并提升可复现性。
- 在测试层集中封装命令执行与断言，避免在 CLI 代码里散落测试逻辑。

## Risks / Trade-offs

- 引入拉取来源覆盖可能被误用 → 仅在测试环境启用，并在文档中明确用途与限制
- 夹具更新与真实模板漂移 → 夹具版本固定，并在变更时同步更新测试
